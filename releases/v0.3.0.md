# Phyllotaxis v0.3.0

## What's New

- **Flags-based CLI** — Commands like `resources`, `schemas`, `auth`, and `callbacks` are now flags (`--resources`, `--schemas`, `--auth`, `--callbacks`) instead of subcommands. This enables composing multiple views in a single call (e.g., `phyll --schemas Pet --auth`).
- **Multi-flag JSON composition** — When multiple flags are used with `--json`, outputs are merged into a single JSON object with keys for each section. Single-flag calls preserve the existing JSON shape for backwards compatibility.
- **External `$ref` resolution** — Multi-file OpenAPI documents with `$ref` pointers to external files are now fully supported. References are bundled into local `#/components/schemas/` entries at parse time.
- **`--context` implies `--expand`** — Using `--context` on an endpoint now automatically expands nested schemas, so you get the full picture without needing to also pass `--expand`.
- **Related schemas shown after expansion** — Schema detail and endpoint context now show "Related schemas" links for any `$ref`s that weren't inlined (e.g., hit the depth limit), so you always know where to drill next.
- **Empty resource groups filtered** — Resource groups with no endpoints (from unresolved tag references) are no longer shown.
- **Empty descriptions suppressed** — Blank or whitespace-only descriptions no longer produce empty lines in output.

## Breaking Changes

- **CLI syntax changed** — All view commands are now flags, not subcommands:
  - `phyll resources` → `phyll --resources`
  - `phyll schemas Pet` → `phyll --schemas Pet`
  - `phyll auth` → `phyll --auth`
  - `phyll callbacks` → `phyll --callbacks`
  - Endpoint detail: `phyll resources pets GET /pets` → `phyll --endpoint GET /pets`
  - `search`, `init`, and `completions` remain subcommands.
- **Terminology renamed** — "spec" → "document" throughout:
  - `--spec` flag → `--doc`
  - `PHYLLOTAXIS_SPEC` env var → `PHYLLOTAXIS_DOCUMENT`
  - Config keys: `spec:` → `document:`, `specs:` → `documents:`
  - Old subcommand names (`resources`, `schemas`, etc.) produce a helpful migration message if used as positional arguments.

## Bug Fixes

- Fixed stale terminology in test assertions, comments, and function names missed during the rename
- Fixed unused variable warning in integration tests

## Upgrade Notes

- Update any scripts or CI that use the old subcommand syntax to the new flag syntax.
- Update `.phyllotaxis.yaml` config files: rename `spec:` to `document:` and `specs:` to `documents:`.
- Update any references to the `PHYLLOTAXIS_SPEC` env var to `PHYLLOTAXIS_DOCUMENT`.
